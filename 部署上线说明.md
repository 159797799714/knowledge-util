# Knowledge-Origin 生产环境部署指南 (CentOS)

## 1. 系统要求

- **操作系统**: CentOS 7/8/Stream
- **Python**: >= 3.11
- **内存**: >= 8GB (建议 16GB+，因涉及 AI 模型加载)
- **存储**: >= 50GB (用于模型缓存、向量数据库等)
- **GPU**: 可选，如有 NVIDIA GPU 可加速向量计算

## 3. 中间件服务部署

### 3.1 安装 Docker & Docker Compose

```bash
# 安装 Docker
sudo yum install -y docker-ce docker-ce-cli containerd.io

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

### 3.2 部署 Milvus (向量数据库)

```bash
# 创建 Milvus 数据目录
mkdir -p ~/milvus-data

# 拉取并运行 Milvus Standalone
docker run -d \
  --name milvus-standalone \
  -p 19530:19530 \
  -p 9091:9091 \
  -v ~/milvus-data:/var/lib/milvus \
  milvusdb/milvus:v2.4.0 \
  milvus run standalone
```

### 3.3 部署 MongoDB (文档数据库)

```bash
# 创建 MongoDB 数据目录
mkdir -p ~/mongo-data

# 拉取并运行 MongoDB
docker run -d \
  --name mongodb \
  -p 27017:27017 \
  -v ~/mongo-data:/data/db \
  -e MONGO_INITDB_ROOT_USERNAME=admin \
  -e MONGO_INITDB_ROOT_PASSWORD=mongo123456 \
  mongo:7.0
```

### 3.4 部署 MinIO (对象存储)

```bash
# 创建 MinIO 数据目录
mkdir -p ~/minio-data

# 拉取并运行 MinIO
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -v ~/minio-data:/data \
  -e MINIO_ROOT_USER=admin \
  -e MINIO_ROOT_PASSWORD=MinIO@202412 \
  minio/minio server /data --console-address ":9001"

# 访问 MinIO 控制台: http://服务器IP:9001
# 创建 Bucket: knowledge-base
```

### 5.3 使用 systemd 管理服务 (推荐)

**创建导入服务 systemd 文件**:

```bash
sudo vim /etc/systemd/system/knowledge-import.service
```

```ini
[Unit]
Description=Knowledge Origin Import Service
After=network.target docker.service
Requires=docker.service

[Service]
Type=simple
User=lighthouse
WorkingDirectory=/home/lighthouse/knowledge-util/knowledge-origin
Environment="PATH=/home/lighthouse/.local/bin:/usr/local/bin:/usr/bin"
ExecStart=/home/lighthouse/.local/bin/uv run python web/api/import_service.py
Restart=always
RestartSec=10
StandardOutput=append:/home/lighthouse/knowledge-util/knowledge-origin/import_service.log
StandardError=append:/home/lighthouse/knowledge-util/knowledge-origin/import_service_error.log

[Install]
WantedBy=multi-user.target
```

**创建查询服务 systemd 文件**:


## 7. Nginx 反向代理 (可选但推荐)

### 7.1 安装 Nginx

```bash
sudo yum install -y nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

### 7.2 配置反向代理

```bash
sudo vim /etc/nginx/conf.d/knowledge-origin.conf
```

```nginx
upstream import_service {
    server 127.0.0.1:8000;
}

upstream query_service {
    server 127.0.0.1:8001;
}

server {
    listen 80;
    server_name your-domain.com;  # 替换为实际域名或 IP

    # 导入服务
    location /import/ {
        proxy_pass http://import_service/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 文件上传大小限制
        client_max_body_size 100M;
        
        # SSE 支持
        proxy_http_version 1.1;
        proxy_set_header Connection "";
        chunked_transfer_encoding off;
        proxy_buffering off;
        proxy_cache off;
    }

    # 查询服务
    location /query/ {
        proxy_pass http://query_service/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # SSE 支持
        proxy_http_version 1.1;
        proxy_set_header Connection "";
        chunked_transfer_encoding off;
        proxy_buffering off;
        proxy_cache off;
    }

    # 健康检查
    location /health {
        proxy_pass http://query_service/health;
    }
}
```

### 10.2 内存不足

```bash
# 增加 swap 空间
sudo dd if=/dev/zero of=/swapfile bs=1G count=16
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 永久生效
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

## 11. 性能优化建议

1. **使用 GPU**: 如有 NVIDIA GPU，将 `BGE_DEVICE` 和 `BGE_RERANKER_DEVICE` 设置为 `cuda:0`
2. **启用半精度**: GPU 环境下设置 `BGE_FP16=True` 和 `BGE_RERANKER_FP16=True`
3. **增加 Worker**: 使用 Gunicorn + Uvicorn 多进程部署
4. **CDN 加速**: 静态资源使用 CDN
5. **数据库优化**: MongoDB 建立索引，Milvus 调整索引参数

## 12. 安全加固

1. **修改默认密码**: MongoDB、MinIO 默认密码必须修改
2. **防火墙限制**: 仅开放必要端口
3. **HTTPS**: 生产环境务必启用 HTTPS
4. **API 认证**: 添加 API Key 或 JWT 认证
5. **定期更新**: 及时更新系统和依赖包
